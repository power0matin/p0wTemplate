# HyperSentry Theme — Professor-Grade Redesign

## Goal
Elevate the hyper-sentry theme from a functional subscription page to a polished, professional showcase piece suitable for academic presentation.

## Files to Modify
- `themes/hyper-sentry/assets/css/main.css` (primary — ~80% of changes)
- `themes/hyper-sentry/assets/js/app.js` (minor — entrance animation triggers, remove time-based theme override)
- `themes/hyper-sentry/index.html` & `sub.html` (minor — add CSS classes, clean inline styles)

---

## Implementation Steps

### Step 1: Extended CSS Token System
Add design tokens to `:root` and `.theme-light` for depth, shadows, spacing grid, typography scale, and timing curves.

New variables:
- **Depth layers**: `--glass-bg-1` through `--glass-bg-4` for layered glassmorphism
- **Shadow system**: `--shadow-sm/md/lg/glow` for consistent elevation
- **8px spacing grid**: `--sp-1` through `--sp-10`
- **Typography scale**: `--text-xs` through `--text-3xl`
- **Timing**: `--ease-out` (cubic-bezier), `--duration-fast/normal/slow`

### Step 2: Typography Refinements
Update font sizes, weights, and letter-spacing across all text elements using the new scale tokens. Key changes:
- Hero value: `--text-3xl` with weight 800, tighter letter-spacing
- Section labels: smaller `--text-sm` with wider `0.08em` tracking
- Body base size: `--text-base` with `line-height: 1.6`

### Step 3: Glassmorphism Upgrade
Replace `.glass-pane` with layered inset borders for depth:
```css
box-shadow:
    var(--shadow-md),
    inset 0 1px 0 rgba(255, 255, 255, 0.05),
    inset 0 -1px 0 rgba(0, 0, 0, 0.08);
```
Add smooth transitions on `box-shadow` and `border-color`.

### Step 4: Spacing Rhythm
Replace all hardcoded margins/paddings with spacing tokens (`--sp-*`) for consistent 8px grid rhythm across header, hero card, stats grid, config list, section titles, and footer.

### Step 5: Entrance Animations (Staggered Reveal)
Add keyframes: `fadeSlideUp`, `fadeSlideLeft`, `scaleIn`. Apply with staggered delays:
- Header: 0s, Profile: 0.1s, Hero: 0.2s, Accordion: 0.3s, Configs: 0.4s, Footer: 0.45s

### Step 6: Micro-Interactions & Hover States
- **Cards**: `translateY(-2px)` lift with enhanced shadow on hover
- **Buttons**: `scale(0.97)` press feedback, glass-btn darkens on hover
- **Stat boxes**: subtle cyan glow border on hover
- **Quick Actions pill**: lift + glow on hover
- **Icon buttons**: scale(0.9) on active, background highlight on hover
- **Theme toggle**: 90deg rotation on active

### Step 7: Progress Bar Polish
- Add shimmer animation (`@keyframes shimmer`) — a light sweep across the fill
- Refined track with thinner 8px height
- Smoother `1.2s` transition using the new ease-out curve

### Step 8: Config Card Stagger Reveal (JS)
In `renderConfigs()`, after `innerHTML` assignment, add JS-driven stagger:
- Each card starts at `opacity: 0; translateY(12px)`
- Animates in with 60ms delay per card

### Step 9: Accessibility
- `:focus-visible` states with cyan outline on all interactive elements
- `@media (prefers-reduced-motion: reduce)` — disable all animations
- Smooth scroll with reduced-motion fallback

### Step 10: Layout Refinements
- **Facet corner**: Add `::after` pseudo-element for a subtle top-edge glow line
- **Mobile**: `@media (max-width: 480px)` — tighter padding, smaller hero value
- **Toast**: Slide-in animation from above

### Step 11: HTML Cleanup
Move inline `style` attributes from `.profile-card` into CSS classes (`profile-label`, `profile-name`).

### Step 12: Code Organization
Add section comment headers (16 sections) for professional code readability.

### Step 13: Remove Time-Based Theme Override
In `app.js` LANG ENHANCER block (lines 273-288), remove the time-based theme logic so `prefers-color-scheme` is respected properly.
